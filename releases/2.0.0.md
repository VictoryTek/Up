# Up 2.0.0

## What's New

### D-Bus Backend Service

Up now ships `up-daemon`, a small privileged service that runs on the system D-Bus and handles all root-level package management operations. This replaces the previous `pkexec sh` persistent shell approach with:

- **Scoped polkit actions** — separate actions for update, cleanup, upgrade, snapshot, and cancel; the daemon only executes pre-approved command patterns
- **Proper cancellation** — cancel any in-flight operation via D-Bus with graceful SIGTERM → SIGKILL escalation
- **Audit logging** — all privileged operations logged to the systemd journal with caller UID/PID and action ID
- **Idle timeout** — daemon exits automatically after 60 seconds of inactivity (socket-activated on demand)
- **Fallback path** — if the daemon is not installed, Up falls back to the legacy `pkexec` path automatically

### Backend Plugin System

Community and distribution-specific package managers can now be added without modifying Up's source code. Place a YAML descriptor in any XDG data directory:

- `/usr/share/up/backends.d/` — distribution-shipped backends
- `$XDG_DATA_HOME/up/backends.d/` — user-installed backends (unprivileged only)
- `/etc/up/backends.d/` — sysadmin overrides

Built-in example descriptors are included for **APK** (Alpine Linux) and **XBPS** (Void Linux).

### Updated Polkit Policy

The polkit policy now uses fine-grained, scoped action IDs:

- `io.github.up.update.system`
- `io.github.up.cleanup.system`
- `io.github.up.upgrade.system`
- `io.github.up.snapshot.create`
- `io.github.up.cancel.operation`

Legacy `io.github.up.pkexec.*` actions are retained for backward compatibility.
