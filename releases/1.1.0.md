# Up 1.1.0

Released: 2026-05-08

## New Features

### fwupd Firmware Backend
A new `fwupd` backend now detects and applies firmware updates via `fwupdmgr get-updates` and `fwupdmgr update`. Firmware updates appear alongside OS and application updates in the main window.

### Pre-Update Snapshot Integration
Up now detects Snapper, Timeshift, and btrfs-native snapshots at startup. Before running a privileged update, an `adw::AlertDialog` offers to create a snapshot. The preference (Always / Ask / Never) is persisted in config.

### Update Changelog Viewer
Each update row now includes a "View Changelog" button. Changelogs are fetched offline (where possible) from the package manager: `apt-cache show` for APT, `dnf updateinfo info` for DNF, `pacman -Si` for Pacman, `zypper info` for Zypper, `brew info` for Homebrew, and `flatpak remote-info --log` for Flatpak. Output is displayed in a scrollable `adw::AlertDialog`.

### Cancel Running Update
A destructive "Cancel" pill button appears while an update is in progress. Clicking it sets a cancellation flag that stops execution after the current backend completes and marks remaining rows as cancelled.

### Disk-Space Pre-Check
Before applying updates, Up estimates the required disk space per backend (`apt-get -s upgrade`, `dnf upgrade --assumeno`, `zypper update --dry-run`, `flatpak remote-ls --updates --columns=download-size`, fwupd JSON `Size` fields). The estimated size is shown in each update row's subtitle. If available disk space (read via `df -k /`) is less than 110% of total required space, a warning dialog is shown before proceeding.

### Scheduled Background Checks
A new `--check` CLI mode runs all backends headlessly (no GTK window), compares the total update count against a stamp file in `$XDG_CACHE_HOME/up/`, and fires a `notify-send` notification when new updates are detected. A systemd user timer (`io.github.up-check.timer`) and one-shot service (`io.github.up-check.service`) are installed via Meson for opt-in daily background checks.

### GNU Gettext Localization Infrastructure
All user-visible strings in the UI are now wrapped with `gettext()` / `ngettext()` calls. Locale initialization (`setlocale`, `bindtextdomain`, `textdomain`) runs at startup. A `po/` directory with `POTFILES.in`, `LINGUAS`, and `po/meson.build` is in place for translators. The Meson build extracts strings and merges the `.desktop` entry template.

## Improvements

### Security & Hardening
- `PrivilegedShell` stdout-sentinel hardened: `\n` and `\0` in command arguments are now rejected to prevent sentinel spoofing.
- `shell_quote` now always single-quotes arguments; the "no quoting needed" fast-path whitelist has been removed.
- `io.github.up.policy` is shipped with scoped polkit actions (`update.system`, `upgrade.system`) instead of relying on the generic `org.freedesktop.policykit.exec` rule.
- Per-command timeouts added to `PrivilegedShell::run_command`; pkexec exit codes 126/127 are surfaced as auth-cancelled errors.
- `LANG=C` enforced on all subprocess invocations used for output parsing.

### Reliability
- `PrivilegedShell` readiness probe now has a timeout; a missing PolicyKit agent is reported diagnostically instead of hanging the UI.
- `rows.borrow()[idx]` panic replaced with by-`BackendKind` lookup in `window.rs`.
- `.expect(...)` panic in `upgrade_page.rs` replaced with a graceful `if let Some` guard.
- Ubuntu upgrade tail-thread leak fixed via `Arc<AtomicBool>` cancellation flag.
- DNF `count_available` now correctly treats only exit code 100 as "updates available".
- `reboot` failures now show a toast notification instead of silently logging to stderr.
- NixOS/Determinate Nix detection works correctly in all environments.
- Flatpak `list_available` uses `flatpak remote-ls --updates --columns=application` for stable, version-independent column layout.
- Fedora `dnf system-upgrade reboot` stdout is now piped to the log instead of discarded.
- Zypper `updated_count` parser uses `LANG=C` and correct line filtering.
- Nix flake target inconsistency fixed: `upgrade_nixos` now uses `resolve_nixos_flake_attr()`.
- Flatpak self-update temp path moved from predictable `/tmp` location to `$XDG_RUNTIME_DIR`.
- Refresh button is disabled while an update is in progress.

### Architecture
- `UpdateOrchestrator` extracted from `window.rs` into `src/orchestrator.rs`.
- `CommandExecutor` trait + `MockExecutor` introduced for testability.
- All backend errors migrated from `String` to `thiserror`-derived enums (`BackendError::{AuthCancelled, Spawn, Exit, Parse, Network, Cancelled}`).
- `Backend::count_available` now has a trait default calling `list_available().map(|v| v.len())`; backends only override when cheap-counting is faster.
- `validate_hostname` / `validate_flake_attr` de-duplicated into a single helper.
- `upgrade.rs` split into `upgrade/check.rs`, `upgrade/version.rs`, `upgrade/execute.rs`.
- Single shared Tokio runtime (`OnceLock<Runtime>` in `src/runtime.rs`) replaces per-spawn runtimes.
- `glib::clone!` macro used throughout UI code to reduce verbose `Rc::clone()` chains.

### Performance
- `LogPanel` buffer capped at 5,000 lines with FIFO eviction.
- Auto-scroll debounced to 50 ms.
- `curl` shell-outs replaced with `ureq 3` for upgrade availability checks.
- Fake progress bar removed; `gtk::Spinner` is the sole in-progress indicator.

### Build & CI
- Release-tag GitHub Actions workflow added with Nix flake artifact upload.
- `meson.build` and `flake.nix` auto-source version from `Cargo.toml`; no more manual version sync.
- `rust-toolchain.toml` added to pin the Rust toolchain for reproducible builds.
- `.editorconfig` added.
- `cargo audit`, `cargo deny`, and `nix flake check` added to `scripts/preflight.sh` and CI.
- Orphaned `cargo-sources.json` deleted.

## Bug Fixes

- Fixed ANSI escape sequences rendering as garbage in the log panel (sequences are now stripped).
- Fixed `LogPanel` growing unbounded during large updates (memory bloat).
- Fixed metered-connection and battery-level warnings not appearing in all cases.
- Fixed reboot-required banner showing even when no reboot is needed.
- Fixed per-backend "skip" checkbox state not persisting across sessions.
