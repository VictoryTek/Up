# Up 1.0.3

**Patch release** fixing Determinate Nix upgrades on non-NixOS distributions (Debian, Fedora, etc.).

## Bug Fixes

- **Fixed Determinate Nix upgrade failing with "determinate-nixd: not found"** — `pkexec` resets `PATH`, so `determinate-nixd` (installed under `/nix/...`) could not be located when the upgrade command was run via `pkexec`; the command now runs unprivileged as intended, since `determinate-nixd` communicates with its privileged daemon via IPC and does not require root on the client side; `needs_root()` no longer returns `true` for Determinate Nix installations
