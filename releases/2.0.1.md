# Up 2.0.1

Released: 2026-05-12

## Bug Fixes

### Nix Build: up-daemon Now Correctly Compiled and Installed

Fixed two compounding bugs that caused the Nix `installPhase` to fail with:

```
install: cannot stat 'target/release/up-daemon': No such file or directory
```

**Root cause 1 — daemon never compiled:**  
`cargoBuildHook` was invoked without `--workspace`, so only the root `up` crate was compiled. The `daemon/` workspace member was never built, producing no `up-daemon` binary.

**Fix:** Added `cargoBuildFlags = [ "--workspace" ]` to the `buildRustPackage` derivation in `flake.nix`. This causes `cargoBuildHook` to pass `--workspace` to cargo, compiling all workspace members, and causes `cargoInstallHook` to install all workspace binaries to `$out/bin/`.

**Root cause 2 — wrong install path:**  
The `postInstall` script referenced `target/release/up-daemon`, which never exists in a nixpkgs build because `cargoBuildHook` always passes `--target <arch-triple>`, placing artifacts under `target/<arch-triple>/release/`.

**Fix:** Updated `postInstall` to move `up-daemon` from the `$out/bin/` location where `cargoInstallHook` places it to the correct `$out/libexec/up-daemon` destination.

Reproduced on: NixOS 25.11, nixpkgs 8fd9daa3, Up 2.0.0.
