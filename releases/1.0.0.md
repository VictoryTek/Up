# Up 1.0.0

**First stable release of Up** — a modern system update & upgrade app for Linux, built with GTK4 and libadwaita.

## Highlights

- One-click **Update All** across every package manager on your system
- Live streaming terminal output for every update operation
- Guided **distro version upgrade** flow with prerequisite checks and backup confirmation
- Native GNOME look and feel via libadwaita

## Features

### Update Mode

- Automatically detects installed package managers at launch
- Runs all detected backends in parallel with per-backend progress bars
- Streams real-time command output to a collapsible log panel
- Prompts to reboot when required after updates

### Supported Backends

| Backend | Description |
|---------|-------------|
| **APT** | Ubuntu, Debian, and derivatives |
| **DNF** | Fedora, RHEL, and derivatives |
| **Pacman** | Arch Linux and derivatives |
| **Zypper** | openSUSE Leap and Tumbleweed |
| **Flatpak** | User and system Flatpak remotes |
| **Homebrew** | Linuxbrew packages and casks |
| **Nix** | Nix profiles (flake and legacy channels) |

### Upgrade Mode

Guided distro version upgrades for:

- **Ubuntu** — via `do-release-upgrade`
- **Fedora** — via `dnf system-upgrade`
- **openSUSE Leap** — via `zypper dup`

Each upgrade flow includes:

- Prerequisite checks (packages up to date, sufficient disk space)
- Backup confirmation step
- Live streaming output during the upgrade process

### Security & Architecture

- Privilege escalation via `pkexec` — no persistent root, no sudo caching
- Flatpak sandbox with minimal permissions; host commands via `flatpak-spawn --host`
- Async command execution on worker threads; results streamed to UI via `async-channel`
- Shell injection hardening: all command arguments passed as discrete tokens, never interpolated into shell strings
- Sensitive output filtered from logs

## Packaging

| Format | Notes |
|--------|-------|
| **Flatpak** | GNOME SDK 48, available via Flathub |
| **Nix** | Nix flake (`nix build` / `nix run`) |
| **Source** | Meson build system (`meson setup && meson compile`) |

## Requirements

- Linux (GTK4 + libadwaita 1.5+, GNOME SDK 48 for Flatpak)
- One or more supported package managers installed on the host

## Known Limitations

- Arch Linux does not support version upgrades (rolling release — upgrade tab is hidden)
- NixOS system upgrades require manual channel management outside of Up
- Homebrew update count is estimated; exact counts depend on brew output format
